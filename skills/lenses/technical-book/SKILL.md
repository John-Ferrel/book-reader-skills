# Technical Book Lens

Use for math, algorithms, engineering, probability, statistics, ML, and software
engineering books.

Focus on definitions, notation, formal objects, theorems, formulas,
algorithms, examples, proof strategy, and method dependencies.

Optional outputs:

- formal-object-index
- notation-index
- method-dependency-map
- example-to-concept map
- source-check queue

First reconstruction minimum:

- notation system addressed or honestly deferred
- formal objects identified when present
- theorem / formula / algorithm inventory when present
- dependency graph or prerequisite map
- examples linked to what they teach
- proof / derivation status labelled
- figures / tables / exercises limitations labelled
- deep-reading queue

If any of these cannot be handled in the first pass, mark them explicitly
deferred with reason. Do not skip them silently.

Read `references/technical-reading.md`.
