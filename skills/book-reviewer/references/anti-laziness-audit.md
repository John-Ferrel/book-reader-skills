# Anti-Laziness Audit

Reviewer must make lazy outputs visible. Do not repair them during review.

Check:

- notes are not merely summaries
- model files use claim cards
- claim cards include Evidence / Confidence / Alternative / What Would Change
- note items include stable IDs
- source anchors are granular enough
- indexes have evidence/status/role and cognitive value
- optional artifacts are useful or correctly marked
- selected lenses handled their core objects
- source limitations are honest
- external context is separated
- reader did not claim complete without independent review
- reviser did not revise without re-reading source

Artifact judgment:

- useful
- redundant
- empty-shell
- premature
- duplicated
- not-applicable
- deferred
- blocked

Action:

- keep
- populate
- merge
- move-to-guide
- defer
- remove
- collapse

Artifact redundancy is reviewer-judged, not creator-self-certified.
