# Knowledge Organization

Indexes are navigation and audit surfaces, not checklist files.

Core indexes:

- concept index
- claim-evidence index
- entity/method/motif index
- design-principle index when applicable

Useful index entries connect model claims, note items, and source blocks. If an
index is not useful yet, mark it deferred/blocked/not-applicable with a reason.

Threads record integration paths across units: concepts, claims, methods,
motifs, entities, examples, or tensions. They should help revise `model/`.
